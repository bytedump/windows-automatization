========================================
 COMANDOS DO TESTE NO SANDBOX
========================================

Fluxo: PASSO 0 (no Windows) abre o Sandbox. Dentro dele, o bootstrap.ps1
RODA SOZINHO no login: pre-cria setupadmin -> roda setup.ps1 -Unattended
(headless, sem GUI) -> imprime no final  RESULTADO: PASSOU / FALHOU.
Os PASSOS 1-3 so sao necessarios pra RE-RODAR a mao dentro do Sandbox.


========================================
 PASSO 0 — NO WINDOWS (HOST): abrir o Sandbox
========================================

Pre-requisito (1x): Windows Sandbox ligado
  Ativar recursos do Windows -> Windows Sandbox -> reiniciar

Abrir PowerShell COMO ADMINISTRADOR (cria C:\SandboxTest na raiz do C:):
  Win+X -> "Terminal (Admin)" ou "PowerShell (Admin)"

Rodar o prep (copia tudo, gera o .wsb e abre o Sandbox):

powershell -ExecutionPolicy Bypass -File "\\wsl.localhost\Ubuntu\home\<seu-usuario>\githubProjects\windows-automatization\tests\prep.ps1"

-> O Sandbox abre e o bootstrap.ps1 roda sozinho. Leia o RESULTADO no fim.


========================================
 DENTRO DO SANDBOX (re-rodar a mao)
========================================

Abrir PowerShell: Win+R -> powershell -> Enter
